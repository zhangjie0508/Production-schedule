## 4. 分配设备

### （1）处理异型管订单

- **初始化设备列**。
- 默认异型管机1和异型管机2负荷都是0。
- 过滤出直管、差异化订单。
- 按“材料厚度”降序排序，确保厚的订单优先分配。
- 确认数据类型正确，避免计算出错：
  - 材料厚度 → `float64`（浮点数）
  - 未完成数量 → `float64`（浮点数）
  - 材料材质 → `string`（字符串）
- 遍历订单，分配设备：
  - 如果厚度不在{0.5, 0.75, 0.6, 0.8, 1.0}之内，或者材料材质包含“**不锈钢**”——强制分配到“异型管机2”。
  - 如果同厚度同材质订单已分配——沿用原设备。
  - **厚度 >= 1.0** 或 **异型管机2负载 <= 异型管机1负载** —— 优先选“异型管机2”。

## 5. 排序

### （1）预计交期转换格式 (`convert_due_date`)

- 解析预计交期格式，如 `3.22 15:00`，转换为 `2025-03-22 15:00`。
- 处理：
  - 去空格
  - 修正中文冒号
  - 防止异常日期，解析失败返回 `NaT`。

### （2）交期排序

- 填充 `NaT` 为 `2100-01-01`（确保无交期的订单永远排在最后）。
- 创建“是否有交期”列，标记（1=有交期，0=无交期）。

### （3）组内排序

- **分组依据**：设备、材料厚度、材料材质。
- 创建“组1”列，标识同设备、厚度、材质的订单属于同一组。

### （4）计算组的最早交期

- 组内排除无交期订单，取最小交期（`min`）。
- 确保“组最早交期”是 `datetime`（日期）格式。

### （5）排序逻辑

| 排序字段   | 作用             | 方向 |
| ---------- | ---------------- | ---- |
| 组最早交期 | 组按最早交期排序 | 升序 |
| 材料材质   | 材质一致性       | 升序 |
| 材料厚度   | 厚度一致性       | 升序 |
| 是否有交期 | 有交期优先       | 降序 |
| 交期排序   | 具体交期排序     | 升序 |

- 确保：
  - 相同材质 & 厚度的订单在一起。
  - 有交期的订单优先。
  - 最早交期的组优先。

### （6）判断是否换料

- 设备/材质/厚度发生变化就需要换料。

### （7）异型管机2排序

- 增加列“异型管机1不可生产”（`True` 表示 1 号机无法生产的订单）。
- 排序逻辑：
  - 异型管机1不可生产 → 让 1 号机无法生产的订单优先排。
  - 组最早交期 → 组内最早交期优先。
  - 材料材质 & 材料厚度 → 让相同材质、厚度的订单排在一起。
  - 是否有交期 → 有交期的订单优先。
  - 交期排序 → 具体交期时间排序。
- 重新合并 `df`：
  - 先删除 `df` 里的异型管机2数据。
  - 用 `concat` 方式重新合并 `df_yixing2`（已经排序）。

### （8）更新是否换料

- 只要设备、材料厚度、材料材质发生变化，就标记为“是”。
- `.shift()` 作用：检查与前一行是否相同。

### （9）计算组1 & 组2

- **组1**：同设备、同厚度、同材质的订单分为一组（`ngroup()` 赋值）。
- **组2**：基于组1进行编号（`factorize()`），序号从0开始连续排序。

## 6. 计算生产时间、生产开始时间、生产结束时间

### （1）计算生产时间 (`calculate_production_time`)

- 不同设备的生产速率：
  - 直管机 → 90件/小时
  - 异型管机1 → 50平方/小时
  - 异型管机2 → 80平方/小时
- 计算出生产总时间并格式化为“X小时 Y分钟”。

### （2）获取下一个可用的工作班次 (`get_next_available_shift`)

- 遍历 `work_shifts` 找到当前时间最近的可用班次。
- 支持跨天计算（如 “21:00 - 08:00” 属于跨天）。
- 返回：
  - 班次开始时间
  - 班次结束时间
  - 当前班次可用的剩余分钟数

### （3）处理休息时间 (`is_in_break_time`)

- 判断当前时间是否处于休息时间段：
  - “12:00 - 13:30”
  - “17:30 - 18:00”
  - “21:00 - 08:00”
- 如果当前时间在休息时间内，返回休息结束时间，否则返回 `None`。

### （4）生产调度与时间计算

- 核心逻辑：
  - 初始化设备的生产起始时间（默认为 `2025-03-21 08:00`）。
  - 换料处理：如果需要换料（`是否换料 == "是"`），则增加15分钟。
  - 生产时间解析：提取“X小时 Y分钟”转换为总分钟数。
  - 模拟生产时间安排：
    - 休息时间处理：如果 `start_time` 在休息时间，则跳过到休息结束时间。
    - 班次检查：获取当前班次的可用时间，如果时间不足，进入下一班次。
    - 计算当前可用生产时间：如果剩余生产时间 > 当前班次剩余时间 → 拆分生产。
  - 更新数据
    - 生产开始时间 → 第一段生产的 `start_time`；
    - 生产结束时间 → 最后一段生产的 `end_time`。
    - 更新 device_last_end_time，确保设备不会重叠生产。

## 7. 计算项目交付时间

- 使用 `groupby("订单编号")` → 以订单编号分组，
- transform("max")` → 找到该订单所有生产批次的最晚生产结束时间。
- 结果存入新列“项目交付时间”。
- 只保留订单编号和交付时间，去重（一个订单只显示一次）。
- 按交付时间排序。

## 8. 判断是否逾期交付

- `预计交期.isna()` → 没有预计交期，默认算“按时交付”。
- `预计交期 >= 生产结束时间` → 实际生产时间早于或等于交期，否则“逾期交付”。
